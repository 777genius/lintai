# SEC480 / CLAUDE-GIT-RESTORE-PERMISSION

`SEC480` flags shared Claude settings when `permissions.allow` grants blanket `git restore` authority.

## Why It Matters

`git restore` can discard working tree or index state. Granting `Bash(git restore:*)` in shared Claude settings broadens default repository mutation authority.

## Trigger Shape

The rule triggers only when all of these are true:

- the file is a detected Claude settings surface
- the path is not fixture-like
- `permissions.allow` contains the exact token `Bash(git restore:*)`

## Clean Cases

These stay clean:

- more specific commands such as `Bash(git restore src/lib.rs)`
- settings files that do not grant blanket `git restore`
- fixture-like examples under test or fixture paths

## Example Trigger

```json
{
  "permissions": {
    "allow": ["Bash(git restore:*)", "Read(*)"]
  }
}
```

## Safer Example

```json
{
  "permissions": {
    "allow": ["Bash(git restore src/lib.rs)", "Read(*)"]
  }
}
```

## How To Fix

Remove shared `git restore` permissions or replace them with a narrower reviewed workflow that keeps restore authority under explicit user control.
