# SEC506 / MD-GH-ISSUE-CREATE-ALLOWED-TOOLS

`SEC506` flags AI-native markdown frontmatter when `allowed-tools` grants the exact token `Bash(gh issue create:*)`.

## Why It Matters

`gh issue create` mutates remote issue trackers. Shared frontmatter should not quietly make issue creation a default capability for every agent that loads the file.

## Trigger Shape

The rule triggers only when all of these are true:

- the file is an AI-native markdown instruction surface
- the path is not fixture-like
- `allowed-tools` or `allowed_tools` contains the exact token `Bash(gh issue create:*)`

## Clean Cases

These stay clean:

- more specific read-only issue commands such as `Bash(gh issue view:*)`
- frontmatter without `Bash(gh issue create:*)`
- fixture-like examples under test or fixture paths

## Example Trigger

```yaml
allowed-tools:
  - Bash(gh issue create:*)
  - Read
```

## Safer Example

```yaml
allowed-tools:
  - Bash(gh issue view:*)
  - Read
```

## How To Fix

Replace `Bash(gh issue create:*)` with a narrower reviewed command, or remove shared issue-creation authority from the frontmatter entirely.
